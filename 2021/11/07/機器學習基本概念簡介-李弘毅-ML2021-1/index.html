    什麼是機器學習呢？

    讓機器具備找一個函式的能力。

    例如:

    • 【輸入是一段聲音，輸出是這段聲音的文字】的這一個函式，這個函式顯然非常複雜，沒有辦法使用人類手寫的方式找尋出來，而我們期待機器自動地將此函式找出來。
    • 【輸入是一張圖片，輸出是描述這張圖片的內容】
    • 讓機器下圍棋，【輸入是一張圍棋的現況，輸出是圍棋的下一步】

    機器學習的類別

    • 1. Regression: the function output is a scalar(數值).例如預測未來的 PM2.5 的濃度
    • 2. Classification: given options(classes), the function output the correct one.例如篩選垃圾郵件、Alfa Go 下圍棋
    • 3. Structured learning: create something with structure.(例如一篇文章、一張圖片)

    機器學習如何去尋找一個函式？

    可以分為三步驟，第一步寫出未知參數的函式，利用 domain knowledge 猜測有哪些因素會影響預測，第二步計算預側與實際物差的 loss function，第三部優化未知函式的參數，此三步驟可稱為 training

    例如去尋找預測 YT 的流量（觀看人數），現有的資料為 YT 的後台，但不知道此函式需要多少參數

    step1: write a function with unknown parameters

    也就是去猜測這個未知參數應該長什麼樣子

    例如：

    \[y = b + wx_1\]

    \(y = b + wx_1\)

    帶有未知 parameter 的 function 又稱為 model，上圖的 model 是一種 linear model

    y: 需要去預測的觀看人數。

    x1: 也許是預測是昨日的觀看人數，是基於什麼東西去預測，因為我們已經知道這個值是多少，所以又稱為 feature。

    1. w: 為 unknown parameter，是透過 domain knowledge 去猜測，後續透過資料去把東西給找出來，因為 w 是和 x1 去做相乘，稱之為 weight，b 沒有和 x1 相乘直接加上去的，稱之為 bias。

    step2: define loss from training data

    一個 function，輸入是剛剛定義的 model 的 unknown parameter，輸出是將 unknown parameter 帶入此 function 後，這筆數值是好或不好，此稱為 loss function。

    每一天的誤差平均為 Ｌ，也就是 Loss

    Ｌ (Loss) 有不同的計算方式

    • 利用絕對值差異的，稱為 mean absolute error (MAE)
    • 利用相差平方的，稱為 mean square error(MSE)
    • 如果計算的數值為機率分佈時，會使用 cross-entropy

    因為 L 是自己定義的，也可能為負數，但上述舉例不可能為負。

    step3: optimization (最佳化)

    找一個 weight 和 bias 使得 Loss 值最小，此堂課只會用到 Gradient Descent。

    Gradient Descent

    1. 計算 w 最小值
    • (Randomly) Pick an initial value w0
    • 計算 { w = w0 時，Ｌ對 w 的微分}是多少，也就是切線斜率是多少，為正，表示 w 值越來越高，為負，表示 w 值越來越低。
    • 計算切線斜率，哪邊比較低，就往低的方向跨出一步，步伐大小取決於兩個因素，一為斜率，二為 η /eta/(learning rate)，若斜率越大，跨出步伐越大，斜率越小，跨出步伐越小，
    • η 是自己設定的，若設定越大，每次 update 的數值越大，學習速度就會越快，若越小則相反，而這種機器學習中需要自己設定的東西稱之為 hyperparameters。

    缺少一個 fancy name?!

    在80. 90 年代稱為 neuron ，通過多層計算稱為 neural network，但名稱被搞臭了(期待過高，實際成果卻沒有達到)，現在稱為 hidden layer，經過多層計算稱為 deep，這整套技術稱為 deep leaning。

    deep 也可以用多個 ReLU 來取代，為什麼要將計算變得更深，而不是變得更胖呢？
